# Reply Polisher

Improves an existing reply draft while preserving the user's intended message.

## Use when

- The user already has a reply draft.
- The goal is to improve wording, tone, structure, or professionalism.
- The user wants the reply to sound better before sending.

## Not for

- Writing a brand-new reply from source context only.
- Summarizing a thread.
- Extracting only actions or deadlines.
- Replacing the user's message with a substantially different one unless explicitly asked.

## Workflow

1. Read the existing draft first.
2. Preserve the intended meaning, commitments, and key facts.
3. Improve flow, phrasing, tone, and readability.
4. Remove redundancy and awkward wording.
5. Keep the final result aligned with the recipient and context.

## Writing rules

- Prefer refinement over full rewrite unless the original draft is unusable.
- Do not add new claims that are not supported by the draft or source context.
- Keep dates, requests, and commitments accurate.
- Return only the polished reply unless the user asks for commentary.
