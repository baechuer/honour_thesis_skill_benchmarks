# Email Drafter

Writes a complete professional outbound email from the available context and the user's intended purpose.

## Use when

- The user wants a new email composed from scratch.
- The task starts from the user's goal, notes, recipient, and context rather than from an incoming message that needs a direct reply.
- The main goal is to produce a complete outbound email the user can send with light editing.
- The setting is academic, administrative, workplace, or otherwise more formal than casual chat.

## Not for

- Replying to an incoming message where the main task is to respond directly.
- Polishing an email draft that already exists.
- Summarizing an email thread without drafting a new email.
- Extracting tasks or follow-up actions only.
- Casual chat-style messages that do not need email structure.

## Workflow

1. Identify the outbound email goal: request, update, introduction, follow-up, confirmation, or escalation.
2. Determine the recipient relationship and appropriate tone.
3. Draft a clear email structure:
   - subject
   - opening
   - main message
   - request or next step if relevant
   - close
4. Preserve important dates, commitments, and facts from the source context.
5. Keep the email concise unless the task clearly requires more detail.

## Writing rules

- Do not invent facts, deadlines, or commitments not supported by the context.
- Prefer clear and professional wording unless the context strongly suggests a casual tone.
- If key context is missing, draft a cautious email that asks for clarification rather than guessing.
- Return only the email draft unless the user asks for alternatives or explanation.

## Default structure

Use this structure unless the task clearly calls for a different one:

1. Brief opening that establishes context.
2. Main purpose of the email in the first paragraph.
3. Supporting detail only if needed.
4. Clear request, proposed next step, or confirmation.
5. Appropriate sign-off.

## Example patterns

### Request email

Use when the user needs something from the recipient.

- Subject should make the ask clear.
- State the request early.
- Include relevant context briefly.
- End with a clear action or question.

### Update email

Use when the user is informing the recipient of progress or status.

- Subject should indicate update or progress.
- Lead with the current status.
- Summarise what changed.
- End with any next step, dependency, or expected timeline.

### Follow-up email

Use when the user is checking in after no response or after a previous interaction.

- Briefly reference the prior context.
- Restate the needed point or request clearly.
- Keep the tone polite and non-accusatory.
- End with a simple response prompt or next step.

## Query-to-email examples

### Example 1

User query:

`Write an email asking my tutor for a two-day extension because I was sick this week. Keep it polite and concise.`

Expected email shape:

- Clear subject indicating extension request.
- Short explanation of the reason.
- Specific request with the new proposed timing.
- Polite close.

Example output:

Subject: Request for Two-Day Extension

Dear [Tutor Name],

I hope you are well. I am writing to ask whether I could have a two-day extension for the assignment, as I was unwell earlier this week and have not been able to work at my usual pace.

If possible, I would be very grateful for an extension until [new date].

Kind regards,
[Your Name]

### Example 2

User query:

`Draft an email to my project supervisor updating them that the literature review draft is finished and I will send the methods section next Monday.`

Expected email shape:

- Subject indicating progress update.
- First paragraph states the current completed milestone.
- Brief note on the next milestone and timing.
- Professional close.

Example output:

Subject: Thesis Progress Update

Dear [Supervisor Name],

I wanted to give you a brief update on my thesis progress. I have now completed the current draft of the literature review section.

My next step is to finish the methods section, and I am planning to send that through to you next Monday.

Kind regards,
[Your Name]

### Example 3

User query:

`Write a follow-up email to the admin office checking whether my form has been processed yet.`

Expected email shape:

- Subject indicating follow-up.
- Brief reminder of the earlier request or submission.
- Clear check-in question.
- Polite, non-pushy tone.

Example output:

Subject: Follow-Up on Form Processing

Dear Admin Team,

I hope you are well. I am writing to follow up on the form I submitted on [date], as I wanted to check whether it has been processed yet.

Please let me know if any further information is needed from me.

Kind regards,
[Your Name]
