# Email Thread Summariser

Summarises an email thread into a clear recap of what happened and where things stand.

## Use when

- The user wants a concise summary of an email thread.
- The thread is long, messy, or repetitive and needs to be reduced to the essential discussion.
- The output should explain the current state of the conversation rather than draft a response.

## Not for

- Drafting a new email.
- Polishing an existing email draft.
- Extracting only actions or deadlines when the user primarily needs an action list.
- Responding to the thread.

## Workflow

1. Identify the main topic of the thread.
2. Track the important messages, decisions, requests, and unresolved points.
3. Collapse redundancy and repeated back-and-forth.
4. Summarise the thread in a way that makes the current state easy to understand.
5. Highlight open issues when the thread does not fully resolve them.

## Writing rules

- Focus on the discussion outcome and current status.
- Keep the summary compact and easy to scan.
- Preserve important dates, decisions, and unresolved questions.
- Return only the summary unless the user asks for a different format.
